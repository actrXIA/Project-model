# (1) Model Description

The dark phase curve is a binary indicator showing whether the current time belongs to the dark or light portion of the cycle. It is not a regulatory signal in itself, but it is used internally to determine whether the current phase is permissive for nocturnal feeding bias. It also affects the effective ghrelin or dG threshold through light-phase offsets.

### 1 Ghrelin

`G = G + k_relax * (G_target - G) + fullness_effect - g_insulin_suppr * I`, where `fullness_effect = g_fullness_drive * (fullness_set - F)`. This means that ghrelin is increased when fullness is below a reference level and suppressed when fullness is above that level. Insulin always suppresses ghrelin.

Biologically, this ghrelin curve is intended to represent the hunger-related drive in the model. It is shaped both by day–night regulation and by meal-related feedback. In trigger mode 1, ghrelin level is directly compared with a threshold to determine meal onset. In trigger mode 2, the model instead uses the rate of change of ghrelin.



### 2 Circadian target

The circadian target is the model’s time-varying ghrelin reference trajectory. It is represented by `G_target` and is calculated as a cosine function using `g_mean`, `circ_amp`, and `circ_peak`. This curve does not itself trigger feeding. Instead, it acts as the target toward which ghrelin relaxes over time. When the circadian target rises, ghrelin is pulled upward; when the target falls, ghrelin is pulled downward.

Conceptually, this curve introduces circadian bias into the system. It is the main reason that ghrelin tends to be more permissive for feeding during the dark phase and less permissive during the light phase.



### 3 Meal input & Triggermode

The meal curve is not a continuous physiological signal, but a behavioural output of the model. It represents discrete feeding events. In the code, the model supports three meal generation modes. In `trigger_mode = 0`, meals are imposed according to a predefined schedule(open protobox to change). In `trigger_mode = 1`, a meal is initiated when ghrelin exceeds a threshold under permissive conditions. In `trigger_mode = 2`, a meal is initiated when the rate of ghrelin rise crosses a threshold.

Once triggered, a meal does not occur only at a single instant. Instead, it lasts for a defined duration, controlled by `meal_width`, during which meal input is applied continuously. Early in the meal, a cephalic signal is also generated, which contributes to the fast insulin response. The meal curve is therefore the most direct representation of feeding behaviour in the model, while the other curves represent regulatory signals that influence or respond to it.



### 4.Fullness

Fullness is represented by `F` and is intended to capture postprandial satiety-related feedback. It is not driven directly by meal events alone, but through an intermediate digestive buffer variable `D`. First, meal input increases `D`, and then `D` drives fullness upward. In the current implementation, fullness is updated as `F = F + fullness_gain * D - fullness_decay * (F - fullness_base)`. This means that fullness rises after meals, but when there is no feeding it decays back toward a non-zero baseline, `fullness_base`, rather than simply returning to zero.

When fullness is below `fullness_set`, it promotes ghrelin recovery. When fullness is above `fullness_set`, it suppresses ghrelin. In addition, fullness contributes to the slower insulin component through the term `slow_drive = F / (insulin_halfmax + F)`. This means fullness influences ghrelin both directly and indirectly through insulin



### 5 Insulin

Insulin is represented by `I` and consists of three components: a circadian baseline component, a fast postprandial component, and a slower postprandial component. The baseline term, `I_base_dynamic`, is a cosine-based circadian function determined by `insulin_base`, `insulin_circ_amp`, and `insulin_circ_peak`. The fast component, `I_fast`, is generated by the cephalic signal at meal onset. The slow component, `I_slow`, is driven by fullness through a saturating function. The final insulin curve is the sum of these three contributions.

Functionally, insulin represents a postprandial metabolic suppression pathway. Compared with fullness, it is less directly tied to gastric filling and more tied to metabolic feedback. In the current model, insulin always suppresses ghrelin through the term `- g_insulin_suppr * I`



### 6 Dark phase(Time period)

The dark phase curve is a binary indicator showing whether the current time belongs to the dark or light portion of the cycle. It is not a regulatory signal in itself, but it is used internally to determine whether the current phase is permissive for nocturnal feeding bias. It also affects the effective ghrelin or dG threshold through light-phase offsets.

In the model the default dark phase starts in 0 to 420 and 1140 to 1440.



# (2)Explanation of the parameters



## Runtime / initial state

- **Run (min)**: total simulation length in minutes.
- **Warmup (d)**: number of warmup days before plotting.
- **G start**: initial ghrelin value.

## Ghrelin / circadian

- **G base**: average circadian target level of ghrelin.
- **G circ amp**: amplitude of ghrelin circadian rhythm.
- **G peak min**: time of ghrelin circadian peak.
- **G relax**: how fast ghrelin moves toward its circadian target.

## Trigger / meal initiation

- **Trig mode**: meal trigger rule. `0` = fixed schedule, `1` = ghrelin threshold, `2` = dG threshold.
- **G trig lvl**: ghrelin threshold for mode 1.
- **dG trig**: rate-of-change threshold for mode 2.
- **Pause min**: refractory period after a meal.
- **F cap**: fullness ceiling that blocks new meals.
- **Trig meal**: size of a triggered meal.
- **Day +G**: extra ghrelin threshold during light phase.
- **Day +dG**: extra dG threshold during light phase.

## Digestive / fullness

- **Digest +**: meal input into digestive buffer.
- **Digest -**: decay rate of digestive buffer.
- **Full +**: how strongly digestive buffer raises fullness.
- **Full -**: how fast fullness decays.
- **F base**: baseline fullness level.

## Insulin baseline / circadian

- **I base**: baseline insulin level.
- **I circ amp**: amplitude of insulin circadian rhythm.
- **I circ pk**: time of insulin circadian peak.

## Cephalic / fast insulin

- **Ceph wid**: duration of cephalic phase after meal onset.
- **Ceph +**: strength of cephalic signal generation.
- **Ceph -**: decay of cephalic signal.
- **I fast +**: how strongly cephalic signal raises fast insulin.
- **I fast -**: decay of fast insulin.
- **I fast sat**: saturation constant for fast insulin input.

## Slow insulin

- **I slow +**: how strongly fullness raises slow insulin.
- **I slow -**: decay of slow insulin.
- **I halfmax**: fullness level giving half-max slow insulin drive.

## Ghrelin coupling terms

- **F -> G drive**: strength of fullness effect on ghrelin.
- **F set**: reference fullness level. Below it, fullness promotes ghrelin; above it, fullness suppresses ghrelin.
- **I -> G sup**: strength of insulin suppression on ghrelin.
